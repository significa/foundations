## Tier 3 — CI validation of `page.mdx` deps  ← next

Build-time script that fails CI if a component imports a sibling not declared in its `page.mdx`.

- Convention: declare deps in `page.mdx` frontmatter:
  ```yaml
  dependencies:
    - components/slot
    - ui/popover
    - ui/divider
  ```
- Script:
  - Walk `src/foundations/<cat>/<name>/<name>.tsx`
  - Extract `@/foundations/...` imports
  - Skip utilities defined in `setup.mdx` (parse `setup.mdx` itself to get the canonical list — one source of truth)
  - Compare to frontmatter `dependencies`
  - Fail if drift
- Add to `pnpm check` pipeline

## Tier 1 — Remaining (last)

### `<Field.Control>` + form spacing + `Label` move

- Add `Field.Control` (Slot wrapper) reading `useField()`, injecting `id` + ARIA attrs into child
- Move `Label` → `Field.Label` (doesn't earn a top-level export)
- Solve form spacing in `<Field>` itself: flex column with intrinsic gap. No more `<Label className="mt-2">`.
- Final API:
  ```tsx
  <Field>
    <Field.Label>Email</Field.Label>
    <Field.Control><Input /></Field.Control>
    <Field.Description>...</Field.Description>
    <Field.Error />
  </Field>
  ```
- Delete `examples/input.tsx`, `examples/textarea.tsx`, `examples/checkbox.tsx`, `examples/radio.tsx`, `examples/select.tsx`, `examples/label.tsx` — they were workarounds
- **Open question**: investigate first — does the project lean on `className` overrides, or is a `gap` prop more idiomatic for layout-ish components?
