# Theming

We opted to use [Tailwind CSS](https://tailwindcss.com/) because of its flexibility and utility-first design. Given that this project is focused on providing foundational code, we've kept the configuration minimal and straightforward.

## Colors

For the color system, we've defined three key color tokens: _background_, _primary_, and _accent_. These colors, combined case-specific alpha values, are used consistently and exclusively across the components.

Please be aware that, theme-wise, copying a component into your project will rarely be a plug-and-play solution: you will often need to adapt the component to match the theme and design of your existing project.

## Utilities

We use two libraries, [`clsx`](https://github.com/lukeed/clsx#readme), [`tailwind-merge`](https://github.com/dcastil/tailwind-merge#readme) and [`class-variance-authority`](https://cva.style/docs), to make working with Tailwind classes easier and more efficient:

- `clsx` helps conditionally apply class names. It lets you easily add or remove Tailwind classes based on component props or state.
- `tailwind-merge` works with `clsx` to automatically resolve any conflicting Tailwind classes, like when you apply different padding or margin values. It ensures that the right styles are applied without conflicts.
- `class-variance-authority` is useful for managing variant-based class combinations, making it easier to handle dynamic styles by defining reusable variants for components.

We combine both `clsx` and `tailwind-merge` in a custom `cn` utility, imported from `lib/tailwind`, to simplify class management into a single function. Additionally, we have a `cnva` utility that brings together all three tools — including `class-variance-authority` — for handling class variants when needed.

### Install

```bash copy
npm install clsx tailwind-merge class-variance-authority
```

### Example

```tsx copy /cn/
import { cn } from 'lib/tailwind';

export function Component({ disabled, className }) {
  return (
    <div
      className={cn(
        'relative text-primary p-12',
        disabled ? 'cursor-not-allowed' : 'cursor-auto',
        className
      )}
    />
  );
}
```

```tsx copy /cnva/
import { cnva } from 'lib/tailwind';

const componentStyles = cnva('relative text-primary', {
  variants: {
    size: {
      md: 'h-12 px-4 text-md',
      sm: 'h-8 px-2 text-sm'
    }
  }
});

export function Component({ size, className }) {
  return <div className={componentStyles({ size, className })} />;
}
```
