# Theming

We opted to use [Tailwind CSS](https://tailwindcss.com/) because of its flexibility and utility-first design. Given that this project is focused on providing foundational code, we've kept the configuration minimal and straightforward.

```js filename="tailwind.config.cjs" copy
const plugin = require('tailwindcss/plugin');

const colors = {
  // in oklch color space
  '--color-background': '97.72% 0 0',
  '--color-primary': '25% 0 0',
  '--color-accent': '87.99% 0.17372425704675545 92.92368848780228'
};

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ['./src/**/*.{js,jsx,ts,tsx,md,mdx}'],
  theme: {
    colors: {
      white: '#fff',
      black: '#000',
      background: 'oklch(var(--color-background) / <alpha-value>)',
      primary: 'oklch(var(--color-primary) / <alpha-value>)',
      accent: 'oklch(var(--color-accent) / <alpha-value>)'
    }
  },
  plugins: [
    plugin(function ({ addBase }) {
      addBase({ ':root': colors });
    })
  ]
};
```

## Colors

For the color system, we've defined three key color tokens: _background_, _primary_, and _accent_. These colors, combined case-specific alpha values, are used consistently and exclusively across the components.

Please be aware that, theme-wise, copying a component into your project will rarely be a plug-and-play solution: you will often need to adapt the component to match the theme and design of your existing project.

## Utilities

We use two libraries, [`clsx`](https://github.com/lukeed/clsx#readme) and [`tailwind-merge`](https://github.com/dcastil/tailwind-merge#readme), to make working with Tailwind classes easier and more efficient:

- `clsx` helps conditionally apply class names. It lets you easily add or remove Tailwind classes based on component props or state.
- `tailwind-merge` works with `clsx` to automatically resolve any conflicting Tailwind classes, like when you apply different padding or margin values. It ensures that the right styles are applied without conflicts.

We combine both `clsx` and `tailwind-merge` in a custom `cn` utility, which we import from `lib/tailwind`, to handle class management in one simple function.

### Install

```bash copy
npm install clsx tailwind-merge
```

```ts copy filename="lib/tailwind.ts"
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs)) || undefined;
}
```

### Example

```tsx copy /cn/
export function Component({ disabled, className }) {
  return (
    <div
      className={cn(
        'relative p-12',
        disabled ? 'cursor-not-allowed' : 'cursor-auto',
        className
      )}
    >
  );
}
```
