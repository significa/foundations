## Features

- **Responsive Design**: Easily create responsive layouts by observing window breakpoints
- **Predefined Breakpoints**: Built-in hooks for common breakpoints (`useMobile`, `useTablet`, `useDesktop`)
- **Custom Queries**: Support for custom media queries through the base `useBreakpoint` hook
- **SSR Compatible**: Works seamlessly with server-side rendering
- **Type Safe**: Full TypeScript support

The hook returns a boolean value indicating whether the window matches the given query.
That is some base breakpoints already available:
- `useMobile`
- `useTablet`
- `useDesktop`

## Examples

### Basic

```tsx
const Example = () => {
  const isMobile = useMobile();

  return (
    <div>
      {isMobile ? <MobileComponent /> : <DesktopComponent />}
    </div>
  );
};
```

### Custom Query

```tsx
const Example = () => {
  const isLargeScreen = useBreakpoint("(min-width: 1200px)");

  return (
    <div>
      {isLargeScreen ? <LargeScreenLayout /> : <DefaultLayout />}
    </div>
  );
};
```

### Multiple Breakpoints

```tsx
const Example = () => {
  const isMobile = useMobile();
  const isTablet = useTablet();
  const isDesktop = useDesktop();

  return (
    <div>
      {isMobile && <MobileLayout />}
      {isTablet && <TabletLayout />}
      {isDesktop && <DesktopLayout />}
    </div>
  );
};
```

## Best Practices

1. **Breakpoint Consistency**:
   - Use the predefined hooks (`useMobile`, `useTablet`, `useDesktop`) for consistent breakpoints across your application
   - Document any custom breakpoints used in your project

2. **Performance**:
   - Avoid using multiple breakpoint hooks in the same component when possible
   - Consider using css or tailwind media queries for simple responsive styles

3. **Server-Side Rendering**:
   - Handle the initial render state appropriately since window size is not available during SSR
   - Consider using a default layout that works well for all screen sizes
