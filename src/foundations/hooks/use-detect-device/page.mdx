## Features

- **Device Detection**: Detect if the user is on a mobile or desktop device
- **Platform Detection**: Identify specific platforms (Android, iOS)
- **SSR Support**: Handles server-side rendering gracefully
- **Type Safe**: Full TypeScript support

## API Reference

The hook returns an object with the following methods:

## Examples

### Basic Usage

```tsx
const Example = () => {
  const { isMobileDevice, isDesktopDevice } = useDetectDevice();

  return <div>{isMobileDevice() ? <MobileLayout /> : <DesktopLayout />}</div>;
};
```

### Platform-Specific Features

```tsx
const Example = () => {
  const { isAndroidDevice, isIosDevice } = useDetectDevice();

  return (
    <div>
      {isAndroidDevice() && <AndroidSpecificFeature />}
      {isIosDevice() && <IosSpecificFeature />}
    </div>
  );
};
```

### SSR Handling

```tsx
const Example = () => {
  const { isSSRDevice } = useDetectDevice();

  return <div>{!isSSRDevice() && <ClientOnlyComponent />}</div>;
};
```

## Best Practices

1. **Initial Render**:

   - Handle the initial render state appropriately since device detection happens after mount
   - Consider using a default layout that works well for all devices

2. **Performance**:

   - Call the detection methods only when needed, as they perform string matching
   - Cache the results if you need to use them multiple times in the same render

3. **SSR Considerations**:
   - Always check for SSR before accessing browser-specific features
   - Provide fallback content for SSR environments

## Technical Notes

- The hook uses the `navigator.userAgent` string to detect devices
- All detection methods are functions to ensure they always return the latest state
- The hook initializes with safe default values (all false) to handle SSR
