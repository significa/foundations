Here's a guide on how to write markdown content using GitHub Flavored Markdown (GFM).

Regular paragraph text with **bold text**, _italic text_, and **_bold italic text_**.

You can also use ~~strikethrough~~ and <u>underlined text</u>.

## Tables

| Prop       | Type      | Default   | Description                                                                                |
| :--------- | :-------- | :-------- | :----------------------------------------------------------------------------------------- |
| `variant`  | `string`  | `primary` | The visual style variant of the button - can be 'primary', 'secondary', 'outline', 'ghost' |
| `size`     | `string`  | `md`      | The size of the button - can be 'sm', 'md', 'lg'                                           |
| `disabled` | `boolean` | -         | Whether the button is disabled and cannot be interacted with                               |
| `loading`  | `boolean` | -         | Whether to show a loading spinner inside the button                                        |
| `asChild`  | `boolean` | -         | Whether to merge props onto a child element instead of creating a button element           |

### Code blocks

You can:

- Choose language
- Highlight lines
- Highlight words
- Show line numbers (and choose starting line with `showLineNumbers{100}`)
- Add a title
- Add a caption

```js {6,7} showLineNumbers "exports" title="tailwind.config.js" caption="footer here"
module.exports = {
  theme: {
    extend: {
      keyframes: {
        marquee: {
          "0%": { transform: "translateX(0%)" },
          "100%": { transform: "translateX(-100%)" },
        },
      },
    },
  },
  plugins: [require("tailwindcss-animate")],
};
```

## Some components

### PropsTable

Renders a table of props for a component.

<PropsTable
  definitions={{
    variant: {
      type: '"neutral" | "success" | "error" | "warning" | "info"',
      default: "neutral", // optional
      description: "The visual style variant of the badge", // optional
    },
  }}
/>

### Button

A simple button component.

<Button>Click me</Button>

### Preview

Renders a preview file.
Will include a button to see the source code.

<Preview file="preview-slug" />

### SourceCode

Renders a code block with the source code of any file.
Can be expandable:

- `expandable` - if true, the code block will be expandable
- `withTitle` - if true, the filename will be displayed as the title of the code block

<SourceCode file="path/to/source/file.tsx" expandable withTitle />

### FileTree

<FileTree initialPath="components/foo.tsx">
  <FileTreeFile name="top.tsx">
    <SourceCode file="path-to-file.tsx" />
  </FileTreeFile>
  <FileTreeFolder name="components">
    <FileTreeFile name="foo.tsx">
      <SourceCode file="path-to-file.tsx" />
    </FileTreeFile>
    <FileTreeFile name="bar.tsx">
      <SourceCode file="path-to-file.tsx" />
    </FileTreeFile>
  </FileTreeFolder>
</FileTree>
